# Implementation Plan

- [x] 1. 実行コンテキストと会話ブロックの基盤を整える
- [x] 1.1 CLI入力の前提条件を検証し、モード別に必須入力を判定する
  - プロフィールは入力が無くても開始し、空入力から対話で情報収集する
  - 求人/評価は必須入力が揃っていない場合は開始前に明示的なエラーを返す
  - 読み取り不能な入力は実行を中断し、部分的な結果を出さない
  - _Requirements: 1.2, 1.3, 1.4_

- [x] 1.2 入力の正規化と実行コンテキストを構築する
  - 入力の空値を除外し、順序を保持したまま正規化する
  - 実行モードごとのコンテキストに格納し、未指定オプションは既定値で補完する
  - _Requirements: 1.1, 1.5_

- [x] 1.3 会話ブロックを一元管理し、時系列と役割を保持する
  - 役割はユーザー入力とエージェント応答の2種類に限定する
  - すべての入力と応答を時系列順に記録し、必要に応じてメタ情報を付与する
  - 役割の追加は要件更新が必要であることを運用ルールとして明示する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.6_

- [x] 1.4 入力が存在しない場合の前提エラーを統一する
  - 入力が必須のモードでユーザー入力が無い場合は実行前にエラーを返す
  - エラー内容はユーザーが不足情報を理解できる形で返す
  - _Requirements: 2.5_

- [x] 1.5 コアロジックの呼び出し口をCLIから分離する
  - コアのツール/エージェント機能を外部から直接呼び出せる入口として提供する
  - CLI はその入口を呼び出すアダプタとして振る舞う
  - _Requirements: 1.6_

- [x] 1.6 Python 実行環境を準備する
  - Python 3.14 を前提に、実行・テストが可能な環境を整える
  - 依存管理は uv を利用できる状態にする
  - _Requirements: 1.6_

- [x] 2. ワークフロー基盤を構築する
- [x] 2.1 最小実行経路（入力収集→検証）を必ず通る構成にする
  - 最小構成で実行可能な経路を用意し、未実装ステップは警告として扱う
  - ワークフロー開始前に必要な前提条件が満たされていない場合はエラーで停止する
  - _Requirements: 3.1, 3.2_

- [x] 2.2 ステップごとの状態更新範囲を限定する
  - 各ステップは担当する状態のみ更新し、他の状態は不変とする
  - 状態スキーマは明示し、更新の責務が分かるようにする
  - _Requirements: 3.3_

- [x] 3. 保存・バックアップ・ログの共通基盤を整える
- [x] 3.1 上書き制御とバックアップを実装する
  - 既存データの上書き禁止時は保存を中断する
  - 上書きが許可されている場合は保存前にバックアップを作成する
  - 保存先が無い場合は作成するか、明確なエラーを返す
  - _Requirements: 4.1, 4.2, 4.5_

- [x] 3.2 実行ログを所定の保存先に残す
  - 質問・警告・入力参照を含めてログを記録する
  - 機微情報はマスク/ハッシュ方針に従う
  - _Requirements: 4.3_

- [x] 3.3 成果物を人間可読な UTF-8 形式で保存する
  - プロフィール/求人/評価の成果物を読みやすい形式で保持する
  - _Requirements: 4.4_

- [x] 4. プロフィール生成と更新のライフサイクルを実装する
- [x] 4.1 入力から構造化プロフィールを生成する
  - メタデータ、要約、経歴、プランのセクションを持つ構造化データを作る
  - 年齢は年齢帯、住所は都道府県レベルの情報として保持する
  - 資格情報は文字列リストとして保持する
  - 型変換に失敗した値はエラーにせず空として扱う
  - _Requirements: 5.1, 5.5, 5.16, 5.17_

- [x] 4.2 欠損検出とドラフト/完了保存を行う
  - 必須項目の欠損は欠損リストとして保持し、即時エラーにはしない
  - 完了判定は経歴の有無で行い、要約は任意とする
  - 欠損が残る場合は未完了として保存し、欠損が無い場合は完了として保存する
  - 既存プロフィールが無い状態の新規作成でも同じ欠損処理で保存する
  - _Requirements: 5.2, 5.3, 5.4, 5.15, 5.18_

- [x] 4.3 対話補完フローを実装する
  - 欠損項目がある場合に質問を用意する
  - 初期入力が空でも対話を開始し、同じ欠損処理で進める
  - 回答を会話履歴に記録し、欠損リストを再評価する
  - ユーザー終了や試行上限到達時は未完了として保存する
  - _Requirements: 5.6, 5.7, 5.8, 5.9_

- [x] 4.4 更新モードの再生成とマージを実装する
  - 更新時は既存プロフィールを必ず読み込む
  - 更新対象を指定された範囲に限定し、その他は保持する
  - 既存が無い場合はエラーで停止し、未知の対象は警告の上で方針に従う
  - _Requirements: 5.10, 5.11, 5.12, 5.13, 5.14_

- [x] 4.5 プロンプト定義を外部化する
  - ProfileAgent の質問プロンプトを設定/テンプレートとして切り出す
  - 実装内のハードコードを排除し、将来の調整を容易にする
  - _Requirements: 5.6, 5.7_

- [x] 4.6 プロフィール対話の質問を外部に返せるAPIを用意する
  - 欠損項目に対する質問一覧を取得できるインターフェースを追加する
  - ループ制御のため、質問のみ返す実行パスを用意する
  - _Requirements: 5.6, 5.7_

- [x] 4.7 CLIでプロフィール対話を実行できる導線を実装する
  - インタラクティブな質問提示→回答受付→再実行のループをCLIで提供する
  - 非対話モードは一度の実行で未完了として保存し、復元は既存プロフィールの読み込みで行う
  - 途中終了・試行上限時は未完了として保存する
  - _Requirements: 5.8, 5.9, 5.15_

- [x] 4.8 対話結果の要約出力をCLIに追加する
  - 完了/未完了の状態と欠損項目を明示的に出力する
  - 保存先と参照情報をユーザーが追跡できる形で表示する
  - _Requirements: 4.3, 5.3, 5.4_

- [ ] 4.9 対話履歴をログ領域に保存する
  - run_id（または実行識別子）を発行し、質問/回答をログに紐づけて保存する
  - ConversationStore の記録結果をログ出力に反映する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 4.3_

- [ ] 5. 求人解析と会社情報補完を実装する
- [ ] 5.1 求人入力を構造化し、会社セクションを分離する
  - 求人入力を読み取り、構造化データを生成する
  - 会社情報が含まれている場合は同一出力内で分離保持する
  - 入力元の参照情報と解析時刻を結果に含める
  - 複数入力を個別に処理し、意図しない上書きを防ぐ
  - _Requirements: 6.1, 6.2, 6.5, 6.6_

- [ ] 5.2 求人入力の欠落や読取不能を明確に扱う
  - 入力が無い/読めない場合はエラーで中断し出力しない
  - _Requirements: 6.4_

- [ ] 5.3 会社情報の補完と外部情報源ポリシーを適用する
  - 補完オプションが有効な場合のみ許可済み情報源から補完を試みる
  - 取得不能時は警告を出して補完せず続行する
  - 外部求人サイトのクローリング/スクレイピングは行わない
  - ユーザー提供の求人テキストは利用規約に従う前提で受け付ける
  - _Requirements: 6.3, 6.7, 6.8, 6.9_

- [ ] 6. 評価と要約のエージェント機能を実装する
- [ ] 6.1 評価結果を生成し保存する
  - プロフィールと求人の両入力を読み込み、スコアと根拠を含む評価結果を生成する
  - 参照元情報を評価結果に含め、保存先は指定がなければ既定値を使う
  - _Requirements: 7.1, 7.3, 7.4_

- [ ] 6.2 入力不備や阻害要因を明確に返す
  - 入力が欠落・不正の場合は部分結果を出さずにエラーを返す
  - スコア計算ができない場合は阻害要因を示す
  - _Requirements: 7.2, 7.5_

- [ ] 6.3 要約/比較を LLM で生成する
  - 自然言語フィルタで対象を選び、重み付けと比較要約を行う
  - 1件のみの場合は比較ではなく要約として扱う
  - _Requirements: 7.6, 7.7_

- [ ] 6.4 要約実行前検証と保存を実装する
  - 不正入力がある場合は LLM 呼び出し前にエラーとする
  - 出所と時刻をプロンプトおよび結果メタに含める
  - 要約/比較結果は既定の保存先に保存し、UI 表示は別スコープとする
  - _Requirements: 7.8, 7.9, 7.10, 7.11_

- [ ] 7. 主要ユースケースのテストを整備する
- [ ] 7.1 代表的な実行フローの統合テストを作成する
  - プロフィール/求人/評価の各フローが期待どおりに連携することを検証する
  - 保存先の生成・バックアップ・ログ記録が行われることを確認する
  - _Requirements: 1.1, 1.3, 1.4, 3.1, 4.1, 4.2, 4.3, 4.5, 5.3, 6.1, 7.1_

- [ ] 7.2 入力欠損やエラー条件のテストを作成する
  - プロフィールは空入力から対話で開始できることを確認する
  - 前提不足エラー、読取不能エラー、評価不能エラーの振る舞いを確認する
  - 欠損が残る場合に未完了として保存されることを検証する
  - _Requirements: 1.2, 2.5, 5.2, 5.3, 5.9, 6.4, 7.2, 7.5_

- [ ] 7.3 要約/比較と会話履歴のテストを作成する
  - 要約/比較でのフィルタ適用、単一要約の分岐を確認する
  - 会話ブロックの順序と役割制約が維持されることを確認する
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.6, 7.6, 7.7_
